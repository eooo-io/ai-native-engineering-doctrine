# Test Design Skill

## Purpose

Create a lean, useful test strategy for a change or component.

## Test types

- Unit tests for pure logic
- Component tests for module behavior
- Integration tests for database, queue, API, or external boundaries
- Acceptance-style tests for user-visible behavior

## Rules

- Prefer meaningful coverage over coverage theater.
- Tests must be runnable by the project tooling.
- Include edge cases and failure cases.
- Avoid brittle tests that assert implementation details without value.
