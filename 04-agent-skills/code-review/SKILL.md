# Code Review Skill

## Purpose

Review a code change and produce actionable feedback.

## Review dimensions

1. Correctness
2. Readability
3. Complexity
4. Security
5. Performance
6. Test quality
7. Architecture fit
8. Operational risk

## Output format

```md
## Summary

## Critical issues

## Suggested improvements

## Tests to add or update

## Questions or assumptions
```

## Rules

- Be specific.
- Explain why each issue matters.
- Prefer fix suggestions or small diffs when possible.
- Do not invent project requirements.
- Mark uncertainty clearly.
