# Job Requirement Matrix

Use this to map job descriptions to harness artifacts.

| Job requirement | Harness artifact | Interview talking point |
|---|---|---|
| Backend systems | Backend standards, API rules, persistence patterns | I define clear service boundaries, tests, and operational expectations. |
| Solution architecture | ADRs, architecture review rubric, pattern catalog | I document trade-offs and choose architecture based on constraints. |
| Agile development | Delivery principles, CI/CD standards | I slice work into reviewable increments with acceptance criteria. |
| AI | Agent operating principles, skill pack, tool adapters | I use AI as governed engineering acceleration, not ad hoc prompting. |
| System design | Architecture patterns, scalability profile | I reason about data, failure modes, scale, and operational cost. |
| Application design | Coding standards, module boundaries | I keep domain behavior out of framework glue. |
| Web applications | TypeScript/Node, API-first, auth standards | I design contracts between frontend, backend, and infrastructure. |
| Cloud Native Design | Docker, Kubernetes, CI/CD | I build deployable systems with health checks, config, and rollback paths. |
| Distributed systems | Event-driven design, observability | I account for retries, idempotency, ordering, and partial failure. |
| Continuous integration | Pipeline gates | I use CI to enforce quality, not merely decorate pull requests. |
| Scalability | Scalability playbook | I scale by measuring constraints before throwing infrastructure at the wall. |
