# Code Review Rubric

## Review categories

| Category | Questions |
|---|---|
| Correctness | Does it do what it claims? Are edge cases handled? |
| Simplicity | Is the solution as simple as the problem allows? |
| Maintainability | Can another developer safely change this later? |
| Security | Does it validate input, protect secrets, and enforce authorization? |
| Performance | Are obvious hot-path problems avoided? |
| Tests | Are meaningful tests added or updated? |
| Architecture | Does it fit existing boundaries and patterns? |
| Operations | Is it observable, deployable, and recoverable? |

## Feedback style

- Be concrete.
- Explain impact.
- Suggest a fix where possible.
- Separate blockers from preferences.
